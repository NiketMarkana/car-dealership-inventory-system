# 01 - Project Requirements

## Objective

Build a full-stack Car Dealership Inventory System following Test-Driven Development (TDD), Clean Code principles, and Software Craftsmanship practices.

## Actors

### Customer

- Register
- Login
- View Vehicles
- Search Vehicles
- Purchase Vehicles

### Admin

- Login
- Add Vehicle
- Update Vehicle
- Delete Vehicle
- Restock Vehicle

## Functional Requirements

### Authentication

- Register
- Login
- JWT Authentication
- Role-Based Authorization

### Vehicle Management

- Add Vehicle
- Update Vehicle
- Delete Vehicle
- View Vehicles
- Search Vehicles

### Inventory

- Purchase Vehicle
- Restock Vehicle

## Non-Functional Requirements

- Secure
- Responsive
- Maintainable
- Tested
- Documented
- Scalable

## Business Rules

- Email must be unique.
- Passwords must be hashed.
- Quantity cannot be negative.
- Price must be greater than zero.
- Only admins can manage inventory.
